# Netlify Deploy Checklist

## Build settings

- **Build command:** `npm ci && npm run build && npm run verify:redirects`
- **Publish directory:** `dist`

## Required deployment artifacts (in `dist/`)

- `index.html`
- `assets/*`
- `sitemap.xml`
- `feed.xml`
- `robots.txt`
- `_redirects` (SPA fallback)

## SPA routing

- BrowserRouter routes are supported by Netlify via `_redirects`:
  - `/`
  - `/blog`
  - `/about`
  - `/herbs`
  - `/compounds`
  - `/contact`
  - `/privacy-policy`
  - `/disclaimer`

## Environment variables

- No required env vars for a basic production build.
- Optional env var:
  - `VITE_LEAD_CAPTURE_ENDPOINT` (or `VITE_NEWSLETTER_ENDPOINT`) to enable live lead/newsletter submissions.

## Custom domain impact

- Custom domain does **not** change build output requirements.
- If domain changes, ensure canonical/site URL settings (where used by sitemap or metadata tooling) are aligned with the production domain.

## What to deploy vs. what not to deploy

- **Deploy:** only the built `dist/` output generated by the build command.
- **Do not deploy directly:** source folders/files such as `src/`, `content/`, `scripts/`, `public/` (source static inputs), and repository metadata/config files.

# Deploy Strategy Report

## Determination

The uploaded `hippie-scientist-patched.zip` is a **mixed bundle** that contains both:

- source-repo files (`src/`, `scripts/`, `package.json`, config files), and
- deploy-ready artifacts (hashed `assets/*`, prerendered route folders, root static outputs like `sitemap.xml`/`feed.xml`).

Because `index.html` references `/src/main.tsx`, the package is **not** a pure deploy-only static bundle.

## Chosen deployment strategy

Adopt **one clean source-repo strategy**:

1. Keep source and build tooling in-repo (`src/`, `scripts/`, `package.json`, `public/`, `content/`).
2. Build in CI (`npm run build`) to produce `dist/`.
3. Deploy only `dist/` via Netlify (`netlify.toml` publish directory).
4. Remove committed root-level deploy artifacts from source control.

## Router decision

Use **BrowserRouter**.

### Why

Current hosting strategy includes SPA fallback (`public/_redirects` with `/* /index.html 200`) and Netlify build config, which supports refresh/deep-link handling for client-side routes. Hash routing is unnecessary with this setup.

## Exact files that should be uploaded/deployed

For deployment, upload/deploy **only the contents of `dist/`** generated by:

```bash
npm run build
```

Do not deploy the repository root directly.

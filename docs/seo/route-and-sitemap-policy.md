# Route and Sitemap Policy

- Canonical route inventory source: `docs/generated/route-inventory.md` (generated by `npm run routes:inventory`).
- Route inventory script: `scripts/ci/generate-route-inventory.mjs`.
- Route SEO validation script: `scripts/ci/validate-route-seo.mjs`.

## Sitemap requirements
- At least one sitemap source must exist: `app/sitemap.ts`, `public/sitemap.xml`, or generated `out/sitemap.xml`.
- Indexable static routes must be in sitemap or intentionally excluded with documentation/warning.
- Dynamic routes included in sitemap must be backed by static params generation.

## Robots requirements
- Canonical robots source is `app/robots.ts` when present.
- Never disallow the whole site.
- Internal/private/testing routes should be explicitly disallowed when they exist.
- Include sitemap URL when sitemap exists.

## Internal link validation rules
- Validate `href` usage in `app/**`, `components/**`, and `lib/**`.
- Allow external URLs (`http`, `https`, `mailto`, `tel`) and anchors.
- Warn on unresolved template-string route hrefs.
- Error on links to missing internal routes.

## Adding a new static page safely
1. Add `app/<route>/page.tsx`.
2. Run `npm run routes:inventory`.
3. Add or confirm sitemap inclusion.
4. Run `npm run validate:route-seo`.

## Adding a new dynamic page safely (static export)
1. Add `app/<route>/[param]/page.tsx`.
2. Export `generateStaticParams` in the page module.
3. Ensure sitemap includes generated paths where indexable.
4. Run `npm run routes:inventory` and `npm run validate:route-seo`.

## Planned or deprecated routes
- Document non-implemented routes as planned/deprecated in docs.
- Do not point navigation links to planned/deprecated routes.
- Keep README high-level; route specifics belong in generated inventory.

# Documentation Index

This directory contains active operating docs plus historical audits. When docs disagree, use this order of authority:

1. `AGENTS.md` for agent rules and route stability.
2. `README.md` for local setup, build, and deployment basics.
3. `docs/site-organization.md` for current site taxonomy and navigation policy.
4. `docs/generated/route-inventory.md` for generated route inventory after running `npm run routes:inventory`.
5. Historical audits and merge reports only as context, not current policy.

## Active Docs

- `docs/site-organization.md` - current public taxonomy, route ownership, and nav policy.
- `docs/build-and-verification.md` - build and verification commands.
- `docs/data-pipeline.md` - workbook-to-runtime data pipeline.
- `docs/generated-data-policy.md` - generated artifact policy.
- `docs/seo/route-and-sitemap-policy.md` - route and sitemap validation expectations.
- `docs/cloudflare-pages.md` - Cloudflare Pages deployment notes.

## Generated Docs

- `docs/generated/route-inventory.md` is generated by `npm run routes:inventory`.
- `docs/internal-link-map.md`, `docs/topic-clusters.md`, and `docs/pages-needing-links.md` are generated by the internal-link engine during data builds.

Do not hand-edit generated docs. Regenerate them from scripts.

## Historical Docs

Audits, merge reports, old plans, and content drafts are retained only when they help explain past decisions. If a historical doc describes routes that no longer exist as active navigation, update its heading to mark it historical or move it under a dated historical note in `docs/audits/` rather than reintroducing a root archive payload.

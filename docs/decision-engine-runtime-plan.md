# Decision Engine Runtime Plan

## Objective

Build a trust-centered decision layer on top of workbook-derived runtime data.

The goal is not maximum content volume.

The goal is:

- faster interpretation
- safer decision support
- transparent uncertainty
- reduced hype
- clearer comparison workflows

## Planned runtime blocks

### Quick Verdict

Short high-signal summary.

Requirements:

- concise
- cautious
- evidence-aware
- never absolute

Example structure:

```txt
Potentially useful for X, but evidence remains limited/moderate and results vary substantially between individuals.
```

## Evidence Confidence

Purpose:

- clarify evidence quality quickly
- reduce false certainty
- support trust differentiation

Possible runtime values:

```txt
Strong
Moderate
Limited
Theoretical
```

## Safety Snapshot

Purpose:

- surface practical caution quickly
- reduce hidden-risk interpretation

Should include:

- major contraindications
- medication cautions
- uncertainty notes
- hormonal or stimulant cautions when relevant

## What We Still Don't Know

Strategic importance:

Very high.

This is one of the strongest trust differentiators available.

Examples:

- lack of long-term human evidence
- conflicting findings
- weak replication
- extract-standardization uncertainty
- poor study quality

## Compare Layer

Purpose:

- reduce decision friction
- avoid aggressive recommendation language
- provide tradeoff-oriented interpretation

Should support:

- focus/cognition
- stress/anxiety
- sleep
- recovery
- longevity

## Runtime rendering constraints

Decision blocks should:

- render from normalized runtime payloads
- avoid freeform enum creation
- fail safely on missing required fields
- preserve uncertainty language

## Anti-hype constraints

The decision engine should never:

- guarantee outcomes
- imply medical treatment
- exaggerate mechanistic evidence
- hide contradictions
- hide safety concerns
- prioritize monetization over interpretation

## Future integration

Future runtime routes may:

- dynamically render compare cards
- generate evidence badges
- generate semantic internal links
- surface contradiction summaries
- render trust-layer UI blocks

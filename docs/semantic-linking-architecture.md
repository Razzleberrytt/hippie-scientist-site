# Semantic Linking Architecture

## Goal

Create a semantic internal-link system that improves:

- navigation clarity
- topical authority
- runtime discoverability
- compare-system usability
- decision-engine rendering

The system should avoid:

- spammy internal linking
- repetitive anchors
- artificial SEO stuffing
- low-context recommendations

## Core principle

Internal links should exist because they improve interpretation.

Not simply because they improve crawl depth.

## Planned semantic relationships

### Compound relationships

Examples:

- synergistic relationships
- compare relationships
- overlapping-use-case relationships
- contradiction relationships
- mechanism relationships

## Topic relationships

Examples:

- sleep
- focus
- stress
- recovery
- longevity
- mood
- motivation

## Runtime opportunities

Future runtime payloads may support:

```txt
related_compounds
related_research_notes
related_compare_tables
related_learn_articles
related_mechanisms
related_outcomes
```

## Decision-layer opportunities

The semantic layer can eventually support:

- compare-card suggestions
- contradiction-aware links
- trust-layer recommendations
- mechanism exploration
- safety-related recommendations

## Anti-spam constraints

The semantic layer should never:

- generate links blindly
- generate excessive repetitive anchors
- prioritize SEO over readability
- bury uncertainty or cautions

## Suggested future scoring

Potential future scoring signals:

- evidence overlap
- mechanism overlap
- user-intent overlap
- safety overlap
- compare-category overlap
- semantic confidence

## Human review importance

Semantic relationships should remain reviewable.

Especially for:

- controversial compounds
- weak evidence areas
- affiliate-sensitive pages
- conflicting evidence topics

## Long-term vision

Eventually the semantic layer may support:

- runtime recommendation systems
- semantic search
- intelligent compare systems
- adaptive decision layers
- contradiction-aware navigation

But the system should remain:

- interpretable
- deterministic
- governance-aware
- trust-centered
